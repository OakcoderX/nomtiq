不是再给你一份榜单，而是帮你选对这一顿

Nomtiq 小饭票让兼容 Agent Skills 的 AI 根据地点、预算、同行的人和用餐场景，实时寻找餐厅，并在本地记住真实反馈。

一句话安装

npx skills add OakcoderX/nomtiq --skill nomtiq -g

OpenClaw 用户也可以通过 ClawHub 安装。需要实时搜索中国大陆餐厅时配置高德 Web Service Key；搜索海外目的地时配置 Serper Key。密钥只应放在环境变量或密钥管理器里。

它适合什么问题？

“今晚吃什么”“附近有什么好吃的”“约会去哪里”“商务请客选哪家”“带孩子聚餐”“一个人吃饭”等需要做餐厅选择的问题。

约会餐厅

考虑安静程度、聊天便利、预算、交通和气氛，不把“最适合拍照”误当成“最适合两个人”。

商务请客

结合人数、包间、停车、正式程度、菜系禁忌和时间安排，减少临场风险。

家庭与多人聚餐

把老人、孩子、忌口、座位和共享菜需求当成明确约束，而不是搜索后的附加项。

它怎么给结果？

Nomtiq 先补齐真正影响选择的信息，再进行实时搜索和约束核对，最后给出“2+1”：两个证据充分的匹配项，加一个明确标注为探索选项的候选。每个结果都说明适合原因、取舍和需要再次确认的信息。

本地口味记忆

吃完后可以记录“菜很好，但环境太吵”一类反馈。口味画像保存在本地，不自动发评论。

示例

北京找一家安静、适合约会、人均 300 元以内的餐厅。不要纯拍照网红店，最好方便聊天。

这类提示同时给出了城市、场景、预算和重要偏好。如果城市范围太大，Agent 应先追问商圈或交通范围；如果实时数据无法确认噪声、价格或营业时间，也必须明确说不确定，而不是编造。

明确的能力边界

小饭票负责餐厅选择和本地反馈记录，不负责菜谱、买菜、热量追踪、外卖下单、餐厅营销、自动发评价，也不会擅自订位或联系餐厅。
